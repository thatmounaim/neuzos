# Bug Report: "Add Group" Button Missing from Settings UI

**Type**: Implementation drift  
**Severity**: High  
**Feature**: 005-session-groups  
**Reported**: 2026-04-25  
**Status**: Patched  
**Patched**: 2026-04-25

## Description

The `addGroup()` function is fully implemented in the script section of `SessionSettings.svelte` (line ~162), but there is **no button in the rendered template that calls it**. The `Card.Footer` area contains only the "Add Session" button and the "Clear All Cache" control. As a result:

1. Users **cannot create groups from the UI** — the group creation code exists but is unreachable.
2. The group dropdown per session row **shows no options** unless groups were pre-seeded in the config file by other means (e.g., config import).
3. From the user's perspective, the Settings > Sessions tab looks like it has a group assignment feature that simply doesn't work — they can see a "Group" column and a dropdown, but the dropdown only ever contains "Unassigned".

The user confirmed: "I can choose a group via dropdown per session, but I don't see any overview or can create groups as wished."

## Artifact Traceability

### spec.md

- **Affected user story**: User Story 1 — Organize Sessions into Groups (P1, MVP)
- **Affected requirement**: **FR-005** — "Settings > Sessions MUST include an 'Add Group' button (in the session card footer) that creates a new empty group with a default name and appends it to the end of the group list."
- **Affected acceptance scenario**: Scenario 1 of User Story 1 — "When the user clicks 'Add Group' in the session card footer, Then a new group with a default editable name appears as a collapsible section above the ungrouped sessions list."
- **Affected success criterion**: SC-001 — "A user with 80 sessions can create, name, and populate 10 groups in under 5 minutes without leaving the Settings > Sessions tab." (Impossible without the Add Group button.)
- **Gap identified**: No spec gap — FR-005 is explicit. This is pure implementation drift.

### plan.md

- **Affected sections**: "Source Code" section — `SessionSettings.svelte` is listed as "MODIFY: add group sections, inline-edit, reorder buttons, delete, group dropdown, **Add Group footer button**". The footer button was not added.

### tasks.md

- **Affected tasks**:
  - **T007** `[X]` — "Add the Add Group footer action and preserve empty groups in `SessionSettings.svelte`" — Marked complete but the button is absent from the template.
- **False completions**: T007 is a false completion. The `addGroup()` helper was implemented but the corresponding UI trigger was omitted.
- **Missing tasks**: No new task is required — T007 must be reopened and its template portion completed.

## Root Cause Analysis

T007 was partially implemented: the `addGroup()` mutation function was added to the script section, but the corresponding `<Button onclick={addGroup}>Add Group</Button>` element was never added to the `Card.Footer` template. The task was prematurely marked complete. The spec (FR-005) and plan both explicitly call for a footer button; it is not present in the rendered output.

## Recommended Fix

1. Run `/speckit.bugfix.patch` to reopen T007 as incomplete.
2. In `SessionSettings.svelte`, add an "Add Group" button to the `Card.Footer` alongside the existing "Add Session" button. Per the spec clarification, it should append a new group with label `"New Group"` to the end of the list.
3. Run `/speckit.bugfix.verify` to confirm consistency.
4. Resume `/speckit.implement` to apply the code fix.
