# UX Report: Session Group Creation Needs Auto-Edit + Drag-and-Drop Consideration

**Type**: UX improvement (not a functional bug)  
**Severity**: Low  
**Feature**: 005-session-groups  
**Reported**: 2026-04-25  
**Status**: Patched (UX-1)  
**Patched**: 2026-04-25

## Description

Two UX friction points were identified in the Settings > Sessions group management flow:

### UX-1 — "Add Group" Creates a Named Group Without Entering Edit Mode

Clicking "Add Group" appends a new group named `"New Group"` but does not automatically place focus in the group's inline name field. The user must then click the group label a second time to rename it. This double-click pattern is unexpected — the most natural expectation when creating a new item is to immediately type its name.

**Root cause**: `addGroup()` in `SessionSettings.svelte` calls `ensureSessionGroups().push({...})` but does not call `startEditingGroup(newGroup)` afterwards. The `startEditingGroup` helper and the label `<Input autofocus>` already exist and support this — the call is simply missing.

### UX-2 — Group Reorder Uses Buttons; No Drag-and-Drop

The spec and plan only require up/down reorder buttons (which are correctly implemented). However, with potentially many groups, drag-and-drop reordering would be significantly more ergonomic. The user explicitly noted "drag and drop seems weird" — interpreted as: the absence of drag-and-drop makes reordering feel clunky.

This is a spec gap, not a spec violation. The spec (User Story 2) defines only button-based reordering and does not prohibit drag-and-drop.

## Artifact Traceability

### spec.md

- **Affected user story**: User Story 1 — Scenario 1: "When the user clicks 'Add Group' … Then a new group with a default editable name **appears**." The spec says the name is editable but does not state the edit mode must auto-open. This is an implicit UX expectation not captured in the spec.
- **Affected success criterion**: SC-001 — "under 5 minutes without leaving the Settings > Sessions tab" — extra clicks add friction that works against this criterion.
- **Gap identified**: Add a note to User Story 1 Scenario 1: "The new group's name field SHOULD be auto-focused for immediate editing so the user can type the desired name without a second interaction."
- **Gap identified**: Add a note to User Story 2 or a new acceptance scenario: "SHOULD support drag-and-drop as an alternative to button-based reorder."

### plan.md

- **Affected sections**: `SessionSettings.svelte` — "Add Group footer button". No mention of auto-focusing the new group's label.

### tasks.md

- **Affected tasks**:
  - **T007** `[X]` — "Add the Add Group footer action" — complete, but auto-focus behavior was not part of the original task description and is now identified as missing.
- **Missing tasks**:
  - A new task: "After `addGroup()` pushes the new group, call `startEditingGroup(newGroup)` so the inline name editor opens immediately."
  - An optional new task (P3): "Add drag-and-drop support for group reordering using a Svelte drag directive or `dnd-action` library."

## Root Cause Analysis

**UX-1**: `addGroup()` was implemented to satisfy the functional requirement (group is created) but the UX expectation (immediate inline edit) was not captured in the task or spec. The wiring is trivial — `startEditingGroup` already exists.

**UX-2**: The spec explicitly chose button-based reorder (P2). Drag-and-drop was not discussed. The user's expectation of drag-and-drop is reasonable for a list management UI.

## Recommended Fix

**UX-1 (immediate)**: In `addGroup()`, capture the new group reference and call `startEditingGroup()`:
```diff
  const addGroup = () => {
+   const newGroup: NeuzSessionGroup = {
+     id: crypto.randomUUID(),
+     label: defaultGroupLabel,
+     sessionIds: []
+   }
-   ensureSessionGroups().push({
-     id: crypto.randomUUID(),
-     label: defaultGroupLabel,
-     sessionIds: []
-   })
+   ensureSessionGroups().push(newGroup)
+   startEditingGroup(newGroup)
  }
```

**UX-2 (future)**: Evaluate adding `svelte-dnd-action` (maintained, Svelte 5 compatible) for group list drag-and-drop as a P3 polish task.
