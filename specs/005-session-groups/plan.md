# Implementation Plan: Session Groups

**Branch**: `005-session-groups` | **Date**: 2026-04-25 | **Spec**: [spec.md](spec.md)  
**Input**: Feature specification from `specs/005-session-groups/spec.md`

## Summary

Add a `sessionGroups: NeuzSessionGroup[]` list to `NeuzConfig` so users can assign named, collapsible groups to their sessions. Groups render as collapsible sections in Settings > Sessions and in the Session Launcher. Group data is exported and imported under the existing `ui-layout` config category. No filesystem or IPC-bridge structure changes are required — all changes stay in types, the renderer `configExport` module, two Svelte components, and the main-process `config.apply_import` / `session_launcher` IPC handlers.

## Technical Context

**Language/Version**: TypeScript 5.x (strict), Svelte 5  
**Primary Dependencies**: Electron 33, electron-vite, Tailwind CSS, shadcn-svelte component library (`Card`, `Table`, `Button`, `Input`, `Popover`, `Separator`)  
**Storage**: JSON flat file at `<userData>/neuzos_config/config.json` — written via `saveConfig()` in main process after every `config.save` IPC call  
**Testing**: No automated test suite in the project; validation is done by manual smoke-testing a release build on Windows  
**Target Platform**: Windows desktop (Electron, primary), Linux desktop (secondary)  
**Project Type**: Desktop application (Electron + Svelte 5 renderer)  
**Performance Goals**: Collapse/expand transitions ≤200 ms perceived; no re-render churn when editing group names  
**Constraints**: Renderer must not bypass IPC bridge; no direct Node.js access in renderer; all new config fields must be backward-compatible (optional with safe defaults)  
**Scale/Scope**: Up to ~100 sessions, ~20 groups; all operations are synchronous and local

## Complexity Tracking

No constitution violations require justification. All changes follow the simplest direct path:
- No new abstraction layers, context providers, or helper modules introduced.
- Group state is stored directly on `NeuzConfig.sessionGroups` and accessed via the existing `neuzosConfig` context in `SessionSettings.svelte`.
- Session Launcher fetches groups via a minimal new IPC handler rather than extending the existing session payload, preserving the existing handler's contract.

**Bugfix**: 2026-04-25 — BUG-001 Session delete must call `config.save` after updating in-memory state so both `sessions` and `sessionGroups` are flushed to disk. The `session.delete` IPC handler only removes partition data; the renderer is responsible for the config write.

**Bugfix**: 2026-04-25 — BUG-002 The `SessionSettings.svelte` `Card.Footer` was missing the "Add Group" button that invokes `addGroup()`. The function was implemented but had no UI trigger. FR-005 and User Story 1 Scenario 1 require this button to be present in the footer.
**Bugfix**: 2026-04-25 — BUG-003 The `session.delete` handler in `src/main/index.ts` targeted `Partitions/<sessionId>` (non-existent); corrected to `Partitions/persist/<sessionId>`. Stop wait increased from 2500 ms to 5000 ms to allow Chromium handle release on Windows before rimraf runs.
**Bugfix**: 2026-04-25 — BUG-004 `addGroup()` in `SessionSettings.svelte` now calls `startEditingGroup(newGroup)` immediately after pushing the new group, so the inline name input auto-focuses without requiring a second click.
**Bugfix**: 2026-04-25 — BUG-005 Mouse button keybinds (Middle, Mouse4, Mouse5) now intercepted in the main process via `app.on('web-contents-created') → webContents.on('before-input-event')`. `NeuzClient.svelte` sends `webview.register_mouse`/`webview.unregister_mouse` IPC on `startClient()`/`stopClient()`. `mouseBindWebContents` Map in `index.ts` tracks active session webview IDs; matched events call `dispatchKeybindEvent(bind)` in main process.
